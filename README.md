main.py

Основной исполняемый файл программного модуля.
Файл отвечает за:
запуск программного приложения;
инициализацию пользовательского интерфейса;
связывание графического интерфейса с бизнес-логикой;
запуск основного сценария работы системы учета заявок.
Файл является точкой входа в приложение.

interface_gui.py

Файл реализует графический пользовательский интерфейс программного модуля.
В файле:
создаются окна приложения;
реализуются формы для работы с заявками;
отображаются списки заявок, пользователей и статистики;
обрабатываются действия пользователя (нажатия кнопок, ввод данных).
Файл отвечает за визуальное представление системы.

interface.py

Файл содержит логику взаимодействия интерфейса с бизнес-уровнем приложения.
Функции файла:
передача пользовательских действий в сервисный слой;
получение данных для отображения в интерфейсе;
обработка фильтрации заявок по ролям и статусам.
Файл служит связующим звеном между интерфейсом и логикой приложения.

services.py

Файл содержит основную бизнес-логику программного модуля.
В файле реализованы:
создание и редактирование заявок;
назначение специалистов;
изменение статусов заявок;
продление сроков выполнения заявок;
бизнес-правила, зависящие от ролей пользователей;
взаимодействие с модулем работы с базой данных.
Файл является ключевым элементом реализации функциональных требований.

database.py

Файл реализует взаимодействие программного модуля с базой данных SQLite.
В файле содержатся функции:
подключения к базе данных;
выполнения SQL-запросов;
добавления, изменения и удаления записей;
получения данных по заявкам, пользователям и ролям;
подготовки данных для отчетов и статистики.

models.py

Файл содержит описания моделей данных, используемых в системе.
В файле определены:
модели пользователей;
модели ролей;
модели заявок;
модели комментариев и статистических данных.
Файл используется для структурирования данных в программе.

stats_module.py

Файл предназначен для расчета статистических показателей.
В нем реализованы:
расчет количества выполненных заявок;
расчет среднего времени выполнения заявок;
формирование статистики по типам неисправностей;
подготовка данных для отображения в отчетах.

import_data.py

Файл предназначен для загрузки данных из внешних файлов в базу данных системы.
Поддерживается импорт:
заявок;
пользователей;
комментариев.
Файл исользуется для первоначального заполнения базы данных.

import_legacy_data.py

Файл реализует импорт данных из ранее используемых форматов.
В процессе выполнения:
данные из CSV, TXT и XLSX файлов преобразуются;
выполняется загрузка данных в текущую структуру базы данных;
обеспечивается совместимость с устаревшими источниками данных.

check_db.py

Служебный файл для проверки корректности структуры базы данных.
Испоьзуется для:
проверки наличия таблиц;
анализа структуры базы данных;
диагностики возможных ошибок в БД.

fix_users_table.py

Служебный скрипт для исправления структуры таблицы пользователей.
Используется при:
изменении структуры таблицы пользователей;
устранении ошибок, связанных с ролями или учетными записями.

reset_requests.py

Файл предназначен для сброса данных по заявкам.
Используется для:
очистки таблицы заявок;
восстановления тестового состояния базы данных;
повторного проведения тестирования.

debug_requests.py

Файл используется для отладки логики работы с заявками.
Выполняется:
тестирование создания и изменения заявок;
проверка бизнес-логики;
анализ корректности работы функций сервиса.

debug_ui.py

Файл предназначен для отладки пользовательского интерфейса.
Используется для:
тестирования отображения форм;
проверки корректности интерфейсных элементов;
анализа реакции интерфейса на действия пользователя.

verify_assignment.py

Файл содержит сценарии проверки назначения специалистов и прав доступа.
Используется для:
тестирования бизнес-прав ролей;
проверки корректности назначения исполнителей;
проверки продления сроков выполнения заявок.

run_tests.py

Файл предназначен для запуска автоматизированных тестов проекта.
Используется для:
проверки основных функций системы;
тестирования логики обработки заявок;
подтверждения корректности работы программного модуля.

setup.py

Файл предназначен для первоначальной настройки проекта.
Исользуется для:
подготовки окружения;
установки зависимостей;
инициализации компонентов системы.
Каталог Кондиционеры_данные
Каталог содержит входные данные для загрузки в базу данных.
